# Design Notes

The wireframes were intentionally low fidelity. The aim was to pin down:

- the main learner path: catalog -> course detail -> completion -> progress
- the main admin path: authoring on the left, oversight on the right
- a simple hierarchy with strong titles, shallow navigation, and visible state

How this informed the final UI:

- the application uses a card-based catalog because it scales well for courses
- course detail prioritizes lesson completion and content readability
- the admin page uses a two-column layout to keep creation and monitoring visible at once
- a warm editorial palette was chosen to avoid a generic default dashboard look
